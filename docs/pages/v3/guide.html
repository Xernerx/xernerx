Welcome to the guide for xernerx V3!

Before you start there is some preconditions that you should know before using xernerx, this being the JavaScript language and the Discord.js library!

Setting up your client

Step 1

The client is the most important thing in xernerx, as it is your initiation of the framework. In order to follow any of the rest of the guide, you need this part.

To start off we'll use the following code. This is just the basic structure, all the following code will be added to this structure.

	import XernerxClient from 'xernerx'; // This imports the client

	new class Client extends XernerxClient {
		constructor() {
			super({
				/* This will be the discord options */
			}, {
				/* This will be the client options */
			}, {
				/* This is your config, and it's optional to parse */
			})
		}
	}

This will only make your client, there is no login yet, and no additional settings set.

Step 2

In step 2 we'll be setting any options required for discord, in the example we're just going to set the intents.

We'll need to get the intents from the Discord.js library, so either get them from the xernerx export, or your own npm install.

	import XernerxClient, { Discord } from 'xernerx';

	new class Client extends XernerxClient {
		constructor() {
			super({
				intents: [ Discord.GatewayIntentBits.Guilds ]
			}, { }, { })
		}
	}

Your code should look like the above code. Import the Discord.js library and use it's intents.

Step 3

So far your bot could sign in, but it won't do anything yet, and to make it easy for you xernerx comes with a built in logger. this can be set in the client options. Which I'll do now. 

	import XernerxClient, { Discord } from 'xernerx';

	new class Client extends XernerxClient {
		constructor() {
			super({
				intents: [ Discord.GatewayIntentBits.Guilds ]
			}, { 
				ownerId: [ "482513687417061376" ], // This will tell the bot that everyone in this array, is an owner of the bot
				logging: true // This will enable the built in logger
			}, { })
		}
	}

When setting the logging on the client to true, it will log that it's online whenever it gets online, that being said your bot has not signed in yet, this will be explained in the next 2 steps.

Step 4 (Optional step)

In this step I'll go deeper on the config object. This step is optional, as you don't require to set your config to the bot.

If you're like me, I like to make a config file in either JSON, or JS modules. This means that you can import your config to any file, it's even easier when the config is provided on the client.

Let's say you have a config file called config.js, and it looks something like the following

	export default {
		token: "YourBotToken",
		prefix: ["?"]
	}

Then you can import this to your main file (where you made your client).

	import XernerxClient, { Discord } from 'xernerx';
	import config from './config.js'; // Import your file

	new class Client extends XernerxClient {
		constructor() {
			super({
				intents: [ Discord.GatewayIntentBits.Guilds ]
			}, { 
				ownerId: [ "482513687417061376" ], // This will tell the bot that everyone in this array, is an owner of the bot
				logging: true // This will enable the built in logger
			}, config)
		}
	} // Set it to the client.

Do as in the code, and your config file is now also found on this.client.config, in pretty much anything to do with xernerx.

Step 5

In this final step we'll continue with finally signing your bot into your bot user. This will also continue on step 3, so keep that in mind for your token!

The way you sign your client in to Discord is very easy.

	import XernerxClient, { Discord } from 'xernerx';
	import config from './config.js';

	new class Client extends XernerxClient {
		constructor() {
			super({
				intents: [ Discord.GatewayIntentBits.Guilds ]
			}, { 
				ownerId: [ "482513687417061376" ], // This will tell the bot that everyone in this array, is an owner of the bot
				logging: true // This will enable the built in logger
			}, config);
		}

		this.register(config.token); // Either import your token (you should do this), or parse it as a plain string.
	}

Congratulations! Your bot can sign in! run the process and it should all work. However no handlers are setup yet!
